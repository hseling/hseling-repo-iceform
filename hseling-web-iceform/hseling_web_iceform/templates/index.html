Formula constructions

Automatic search for formula constructions in Old Norse-Icelandic sagas

The oral formulaic theory described the formula as a building block of oral composition that was strictly connected to meter (Parry 1930, Lord 1960). However, recently there have been several studies discussing formulas in prose traditional narratives (Lamb 2015, Sävborg 2018) and showing that in such prose corpora formulas are used without metrical conditions. The main goal of this project focused on an automatic search for formulas and meaningful repetitions in Old Norse-Icelandic sagas. For more information about the project, see the Project description section.

The main idea of the algorithm is the compilation of a list of ngrams, its further filtration, and application of NLP methods to determine repeating semantically significant word sequences. You can see the results in the Formula search section.

Project description

Meaningful repetitions in sagas can be either formulas found in many sagas and used under similar conditions (for example, the formula of fairy-tale origin "Once upon a time"), or repetitions of phrases found in a particular saga, which often turn out to be a narrative device (for example, in the Hans-Horis saga, the same phrase occurs four times in different variations, thus framing the burning of Blund-Ketill).

The main goal of this project is to automatically search for repetitive structures with the possibility of variability.

Repetitions of this kind are quite variable. Often, some of the words can be replaced by lexical or contextual synonyms. Despite the fact that with the traditional manual search, such constructions are easy to notice, the search itself is very time consuming. For this reason, we were interested in automatisation of the search process and develop an algorithm that could facilitate the task of researchers.

Certain formulas and their variants can appear rarely in the corpus. Therefore, the entity filtering based on frequency is not applicable. Special heuristics were designed to gradually filter examples and combine contextual synonyms in groups. The algorithm consists of four stages:

  • Ngram list formation
  • Ngram grouping
  • Syntactic grouping
  • Semantic clustering
